Porto-UnOrdered-List Documentation

Overview

The Porto-UnOrdered-List component allows you to create customizable unordered lists with various styles, icons, and animations. It supports nested lists and multiple configurations to match your design needs.

For more details, check out the Porto-UnOrdered-List repository on GitHub or the Porto documentation.

Settings

Settings Properties

Property Options Description
List Border
  • list-borders: Adds borders around the list items.
  • list-side-borders: Adds borders only on the sides of the list items.
  • "": No borders (default).
 Defines the border style for the list items.
Icon Style
  • list-icons-style-2: Applies the second icon style.
  • list-icons-style-3: Applies the third icon style.
  • "": Default icon style (no additional styling).
 Defines the style of the icons used in the list.
Icons Position
  • list-icons-reverse: Reverses the position of the icons.
  • "": Default position (icons on the left).
 Defines the position of the icons in the list items.
Icons Sizes
  • list-icons-sm: Small icons.
  • list-icons-lg: Large icons.
  • "": Default icon size.
 Defines the size of the icons in the list items.
Icons Color
  • list-primary: Primary color.
  • list-secondary: Secondary color.
  • list-tertiary: Tertiary color.
  • list-quaternary: Quaternary color.
  • "": Default color.
 Defines the color of the icons in the list items.
Enable Animations
  • true: Enables animations.
  • false: Disables animations (default).
 Specifies whether to enable animations for the list items.
Animation Type
  • fadeInUp: Fades the list items in and moves them up.
 Defines the type of animation for the list items. This property is only available when Enable Animations is set to true.
Animation Delay
  • 0: No delay.
  • 300: 300ms delay.
  • 600: 600ms delay.
  • 900: 900ms delay.
  • 1200: 1200ms delay.
 Specifies the delay (in milliseconds) before the animation starts. This property is only available when Enable Animations is set to true.

Views

Below are examples of the different styles and configurations available for the Porto-UnOrdered-List component. Each style is customizable and can be adapted to your design needs.

Note:
  • When Type is set to Unordered, Unstyled, or Icons, the Items (for Unordered, Unstyled, or Icons) property is used. Each item can include:
    • List Item Text: The text of the list item. (Required)
    • Icon: An optional icon for the list item. Example: fas fa-check.
    • Second Level Items: A nested list of items with their own text and optional icons.
  • When Type is set to Description, the Items (for Description type) property is used instead. Each item includes:
    • Title: The title of the description item. (Required)
    • List Item Text: The text of the description item. (Required)
  • Dependencies: The Items (for Unordered, Unstyled, or Icons) and Description lists properties are mutually exclusive. Only one will be displayed based on the selected Type.
