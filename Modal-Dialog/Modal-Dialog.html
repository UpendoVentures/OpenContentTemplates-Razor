Modal-Dialog Documentation

Overview

The Modal-Dialog component allows you to create customizable modal dialogs with configurable titles, content, buttons, and styles. This component is designed to provide flexibility and control over how modal dialogs are displayed and interacted with. It supports the following features:

  • Configurable Titles: The modal header can display a title with customizable text, color, and HTML heading level (e.g., h1, h2, h3).
  • Dynamic Content: The modal body supports plain text or HTML content, allowing for rich and interactive user experiences.
  • Action Buttons: Includes primary and secondary buttons with customizable text, links, and styles to guide user actions.
  • Cookie-Based Behavior: Optionally, the modal can use cookies to control its display. For example, it can be configured to show only the first time a user visits the page.
  • User-Specific Content: If a user is authenticated, the modal can display personalized information, such as the user's email address.
  • Customizable Styles: The modal's header, buttons, and other elements can be styled to match your application's design using configurable settings like background color, text color, and hover effects.

This component is ideal for scenarios where you need to display important information, collect user input, or guide users through specific actions. For more details, check out the Modal-Dialog repository on GitHub.

Settings

Settings Properties

Property Data Type Description
Header Background Color String Defines the background color of the modal header. Default:#CE3439.
Header Text Color String Defines the text color of the modal header. Default: #FFFFFF.
Header Text Type Select Specifies the HTML heading level for the modal title. Options include h1, h2, h3, h4, and h5. Default: h3.
Use Cookie Setting Boolean If enabled, the modal will only open the first time the user accesses the page. Default: true.

When this setting is enabled, the component uses a cookie named "firstRender" to track whether the modal has already been displayed to the user. Here's how it works:
  • Cookie Name: firstRender
  • Purpose: Prevents the modal from being shown repeatedly by storing a flag in the user's browser.
  • Duration: The cookie is set to expire after 1 day by default.
  • Behavior:
    • If the cookie firstRender does not exist, the modal will be displayed, and the cookie will be created.
    • If the cookie exists, the modal will not be displayed again until the cookie expires or is deleted.

How to Find or Delete the Cookie:
  • Open your browser's developer tools (usually accessible via F12 or Ctrl+Shift+I).
  • Go to the "Application" tab (in Chrome/Edge) or "Storage" tab (in Firefox).
  • Under "Cookies," locate the domain of your website.
  • Search for the cookie named firstRender.
  • To delete the cookie, right-click on it and select "Delete" or use the browser's cookie management tools.

Disabling the Cookie:
  • If you disable the Use Cookie Setting in the module's configuration, the cookie will not be created, and the modal will display every time the page is loaded.
  • To disable the cookie programmatically, the following line in the script ensures the cookie is cleared: 
    document.cookie = "firstRender=; Max-Age=-99999999;";
