Overview

The Porto-Tables component allows you to display organized data in a structured table format with an attractive design.It provides an efficient way to present information in rows and columns, ensuring a clear and professional layout. Porto defines the appearance and behavior, while its implementation is managed through OpenContent, providing flexibility in configuration.

For more details, check out the Porto-Tables repository on GitHub or the Porto documentation.

Properties

Field Name Type Description
Module Title Text The title of the module.
Table Head Array Defines the table headers. Each item contains the column header text.
Table Column Header Text The text for a table column header.
Table Body Array Defines the table rows. Each item represents a row with optional classes and cells.
Row Class List The class applied to a table row. Available options:
  • info: Info
  • danger: Error
  • success: Success
  • warning: Warning
Cells Array Defines the cells within a row. Each item contains the cell content.
Cell Content (required) Text (WYSIWYG) The content of a table cell.

Settings

Settings Properties

Setting Type Description Options
Striped Checkbox Applies striped rows to the table for better readability.
  • true: Enable striped rows.
  • false: Disable striped rows (default).
Bordered Checkbox Adds borders to all table cells.
  • true: Enable bordered table.
  • false: Disable bordered table (default).
Hover Rows Checkbox Highlights rows when hovered over.
  • true: Enable hover effect on rows.
  • false: Disable hover effect (default).
Condensed Checkbox Reduces the padding in table cells for a more compact table.
  • true: Enable condensed table.
  • false: Disable condensed table (default).

Notes

Note: When using the Porto-Tables component, ensure the following considerations are met for proper functionality:
  • Enable or disable table features (e.g., striped rows, borders, hover effects, condensed layout) using the settings provided in the configuration.
  • Ensure all table headers and rows are properly defined to display the desired content.
  • Verify that the table layout adapts correctly to different screen sizes and devices.
  • Test the visual appearance of the table, including hover effects and borders, to ensure it aligns with your website's design.
  • Use valid and accessible content for table cells to maintain usability and compatibility.
  • Check that all settings (e.g., striped, bordered, hover rows, condensed) are applied as expected in the rendered table.

Proper configuration and testing will help ensure that the Porto-Tables component integrates seamlessly into your website and provides a professional and functional table layout.
