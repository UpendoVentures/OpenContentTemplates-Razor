Overview

The Porto-Forms component allows you to integrate customizable forms into your website, making it easy to collect data and interact with users. It offers various customization options, including styles, sizes, and layouts, to fit the specific needs of your interface. Porto defines the appearance and behavior of the forms, while their implementation is managed through OpenContent, providing greater flexibility and control.

For more details, check out the Porto-Forms repository on GitHub or the Porto documentation.

Form Properties

Property Data Type Description Example
Module Title String Used to define the title of the module. This title is displayed in the DNN administration interface to help identify the module.
Module Anchor String Used to create anchor links on the page. This allows users to link directly to the module using a unique identifier. Only letters, numbers, and hyphens are allowed.
FormTitle String Defines the title of the form. "Contact Us"
ButtonText String Defines the text displayed on the form's submit button. "Submit"
Items Array Contains the form fields grouped inside the form. See below for details.

Form Group Properties

Property Data Type Description Example
Item String Defines the label of a form field. "Name"
DataType String Specifies the type of input field. Options: text, email, url, textarea. "text"
GridWidth String Defines the width of the field using Bootstrap grid classes "col-md-6"
Rows String Defines the number of rows for a textarea. Options: 1 to 10. Required when DataType is textarea. "5"

Notes

Note: The Porto-Forms component allows you to create customizable forms to collect data and interact with users effectively. Ensure the following properties are configured correctly to match your design and functionality needs:
  • FormTitle: Defines the title of the form.
  • ButtonText: Specifies the text displayed on the submit button.
  • Items: Contains the form fields, including their labels, types, and layout settings.
Additionally, explore the available settings, such as button colors and grid-based layouts, to align the form with your website's design. Examples of configurations and styles are provided above.
