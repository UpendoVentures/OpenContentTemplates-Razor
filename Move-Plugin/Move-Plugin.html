Overview

Move-Plugin is a reusable template within the OpenContentTemplates-Razor repository that allows modifying the position of third-party iframes (such as chat buttons or other elements added through external scripts). This template enables adjusting the iframe's location by configuring its height, left, right, or bottom position according to specific needs. It also offers the option to hide iframes when not needed. It can be configured to work exclusively in DNN's editing mode or globally, providing flexibility in its implementation.

For more details, check out the Move-Plugin repository on GitHub.

How to use it?

In this session, I will explain step by step how to use the Move-Plugin template, from its installation on the website to its configuration and testing. It is essential to have Open Content installed on the DNN site, as this template relies on that module for proper functionality. By following these instructions, you will be able to integrate it correctly and ensure it works as expected.

Template Installation

Below are the steps to install the Move-Plugin template on a DNN site with Open Content:

  1. Verify Open Content Installation
    • Ensure that the Open Content module is installed on the DNN site. If it is not installed, download and install it from the official page .
  2. Add the Open Content Module to the Page
    • Navigate to the page where you want to use the template.
    • Add a new Open Content module in the desired location.

  3. Configuring the GitHub Repository
    • Edit the newly added Open Content module.
    • Select the option Edit Global Settings
    • Set the repository name in the Github Templates Repository field. In this case, use UpendoVentures/OpenContentTemplates-Razor.
    • Click on Save.

  4. Add the template to the site.
    • Click on Create new template folder.
    • Select the Web (GitHub) option.
    • Select the template you want to add, in this case, Move-Plugin.
    • Click on Create.

  5. Edit the template content.
    • Click on the Edit button of the module.
    • Click on the Edit option from the menu that appears.
    • Set the necessary values for the content you want to display in the modal. Each of the fields is explained in the Edit Properties section.

Notes

Note: When using the Move-Plugin component, ensure the following considerations are met for proper functionality:
  • Important: The Query Selector and Value fields must be configured correctly to target the desired element. An incorrect selector may result in the element not being found or modified.
  • Important: The Modification Type field determines whether the element will be hidden or moved. Ensure this is set to Move Position if you want to adjust the element's position.
  • Move Properties: If you are using the Move up?, Move Left?, Move Right?, or Move Down? options, ensure the following:
    • The corresponding Current Value and New Value fields are set correctly to define the element's current and target positions.
    • If Only in DNN edit mode? is enabled, the changes will only apply when the site is in edit mode. Ensure this is configured according to your requirements.
  • Custom Selectors: When using the Custom option in Query Selector, you can define advanced selectors using JavaScript.

Proper configuration of these fields is critical to ensure the Move-Plugin works as expected. Test thoroughly to verify the desired behavior.
