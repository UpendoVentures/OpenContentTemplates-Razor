Overview

The Modal-Dialog-NoBS is a generic OpenContent Razor template designed for use in projects that require a confirmation modal. Its design is based on custom CSS, allowing it to render correctly without dependencies on frameworks like Bootstrap. It includes a set of properties and configurations that enable customization and adaptation to different environments and requirements.

For more details, check out the Modal-Dialog-NoBS repository on GitHub.

How to use it?

In this session, I will explain step by step how to use the Modal-Dialog-NoBS template, from its installation on the website to its configuration and testing. It is essential to have Open Content installed on the DNN site, as this template relies on that module for proper functionality. By following these instructions, you will be able to integrate it correctly and ensure it works as expected.

Template Installation

Below are the steps to install the Modal-Dialog-NoBS template on a DNN site with Open Content:

  1. Verify Open Content Installation
    • Ensure that the Open Content module is installed on the DNN site. If it is not installed, download and install it from the official page .
  2. Add the Open Content Module to the Page
    • Navigate to the page where you want to use the template.
    • Add a new Open Content module in the desired location.

  3. Configuring the GitHub Repository
    • Edit the newly added Open Content module.
    • Select the option Edit Global Settings
    • Set the repository name in the Github Templates Repository field. In this case, use UpendoVentures/OpenContentTemplates-Razor.
    • Click on Save.

  4. Add the template to the site.
    • Click on Create new template folder.
    • Select the Web (GitHub) option.
    • Select the template you want to add, in this case, Modal-Dialog-NoBS.
    • Click on Create.

  5. Configure the template.
    • Click on the Template settings button.
    • The template configuration options will be displayed, which are described in the Settings section.

  6. Edit the template content.
    • Click on the Edit button of the module.
    • Click on the Edit option from the menu that appears.
    • Set the necessary values for the content you want to display in the modal. Each of the fields is explained in the Edit section.

Settings

Settings Properties

Setting Type Description
Header Background Color Text It allows you to set the background color of the modal dialog title in various formats:
  • hexadecimal #CE3439
  • RGB rgb(206, 52, 57)
  • HSL hsl(356, 60%, 51%)
  • color names like red, blue, darkorchid, etc.
Header Text Color Text Allows you to set the text color of the modal dialog title in various formats:
  • hexadecimal #CE3439
  • RGB rgb(206, 52, 57)
  • HSL hsl(356, 60%, 51%)
  • color names like red, blue, darkorchid, etc.
Verification EndPoint Text Allows you to set the URL of an endpoint to check whether the modal should be shown or not. This endpoint must return a JSON object with a boolean property called showModal.
Confirmation EndPoint Text Allows you to set the URL of an endpoint for the main button action of the modal dialog. This endpoint must return a JSON object with a boolean property called success, which indicates whether the functionality was executed correctly or not. If the value of success is true, the modal will close; if false, the error message defined in the Error message when confirming property will be displayed.
Error message when confirming Text Allows you to set the message to be displayed when the response from the endpoint of the modal dialog's main action is false.
Generic error message Text Allows you to set the text to be displayed when an error occurs in the endpoint calls.
NoUserLoggedIn.Text Text Allows you to set the text to be displayed if the user is not authenticated or if the user's email cannot be loaded correctly.
Non-confirmation message Text Allows you to set the text of the message to be displayed when trying to click the main action button of the modal dialog without checking the confirmation checkbox.

Notes

Note: When using the Modal-Dialog-NoBS component, ensure the following considerations are met for proper functionality:
  • Important: Configure the VerificationEndPoint correctly to determine whether the modal should be displayed. This endpoint must return a JSON object with a boolean property called showModal.
  • Important: Configure the ConfirmationEndPoint correctly to handle the main button action. This endpoint must return a JSON object with a boolean property called success.
  • Ensure that the success property in the ConfirmationEndPoint response is true to close the modal upon successful action, or false to display the error message.
  • Ensure that the showModal property in the VerificationEndPoint response is true to show the modal or false to hide it.

Proper configuration and testing will help ensure that the Modal-Dialog-NoBS component integrates seamlessly into your website and provides a professional and functional table layout.
